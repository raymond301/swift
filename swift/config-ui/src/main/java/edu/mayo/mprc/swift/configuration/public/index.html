Configuration Instructions

Welcome to Swift configuration screen. In order for Swift to work, you need to configure its components.

Swift can run multiple search engines (Mascot, Sequest, Tandem, Omssa and MyriMatch) and combine their results into a single Scaffold file. Swift can run on a single computer, multiple computers or an entire cluster. This page lets you set that all up.

We recommend going through the items in the tree below, one by one.

  1. Create all the daemons you plan to run Swift on
  2. Configure these daemons
    1. provide host name and OS type, this helps setting up the modules
    2. for multi-module setup, do not forget to configure the shared disk space for each of the daemons
  3. Configure the database. The database stores file paths as seen by its containing daemon.
  4. Configure the message broker
  5. Configure the Swift Searcher.
    • As you go through the searcher configuration, create modules for services you want to use.
    • Configure the .fasta directories for the searcher
  6. Configure all the remaining Swift modules
  7. Save the configuration

When the configuration gets saved, the installation will:

  1. create config file conf/swift.conf - it defines what services should Swift run, how and where
  2. run a check, making sure all parts are configured properly
  3. run an installation script, making sure that all requested folders are created, and that the database is seeded with initial data

When you are done configuring, press the Save configuration button below, kill the configuration script and start the provided bin/swift script.

Keep in mind that the daemon containing the embedded Message Broker module has to start first, so other modules can use it to communicate.
